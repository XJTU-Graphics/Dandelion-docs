编译和运行环境

编译和运行的环境必须满足如下条件：

  • 提供支持 C++ 17 标准的编译器、可以运行 CMake
  • 64 位操作系统
  • 硬件（驱动）至少支持 OpenGL 3.3 版本

运行于 x86-64 硬件架构上的 Windows 10 / 11、Mac OS X 和 Debian / Ubuntu / Linux Mint / Arch Linux / Manjaro 等常见 Linux 发行版均满足条件。由于我们没有使用 Apple Scillion 处理器的设备，不保证在 Apple M 系列芯片的计算机上一定运行正常。另外，Virtual Box 支持的 OpenGL 版本太低，如欲使用虚拟机运行，请用 VMware Player / Workstation 创建虚拟机并开启硬件加速选项。

依赖

这个项目依赖于如下的第三方库：

  • GLFW 3.3.8, Zlib License
  • GLAD (OpenGL Core Profile, version >= 3.3), MIT & Apache License
  • Eigen 3.4.0, Mozilla Public License
  • Assimp 5.2.5, Open Assert Import Library License
  • Dear ImGui 1.89.3 (docking branch), MIT License
  • stb_image 2.28, MIT License & Public Domain
  • spdlog 1.11.0, MIT License
  • portable-file-dialogs (commit 7f852d88a480020d7f91957cbcefe514fc95000c), WTFPL License

所有的依赖项均被包含于 deps 子目录中，不需要额外下载或安装。

工具需求

  • 支持 C++ 17 标准的编译器（推荐使用 Visual Studio 2019 及以上版本附带的 MSVC、GNU G++ 10 及以上、Clang++ 12 及以上）
  • CMake 3.8 及以上（推荐使用最新的稳定版本）

在基于 Debian 的 Linux 发行版环境下，请安装 build-essential cmake cmake-extras 软件包。

编译

Windows

Linux

打开终端，进入 dandelion 目录并执行如下命令：

$ mkdir build
$ cd build
$ cmake ..
$ make -j8

调整 cmake 命令的参数可以调整编译设置。修改 make 命令 -j 参数后接的数字可以调整编译器使用的线程数，建议和物理核心数量等同。
